Create, reuse, and repurpose.Teams can assemble and launch n-tier applications by mixing and matching boxes. Launching applications from boxes gives you consistency and agility. Versioning means you can innovate without breaking things.

See Why Boxes Are Awesome

What is a Box you ask? A Box is an encapsulated, fully configured, component of your application architecture. A Box contains scripts, variables, files, and other metadata required to deploy the application. It could be a database, webserver, middleware, language, runtime – really you can Box almost anything. A Box is reusable, can be versioned, and is fully mobile across cloud services.
